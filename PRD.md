# CUDA Kernel Graph化项目 PRD文档

## 1. 项目概述

### 1.1 项目名称
**KernelGraph** - CUDA Kernel 计算图可视化与分析工具

### 1.2 项目背景
CUDA kernel 是 GPU 并行计算的核心组件，但其代码结构复杂、数据依赖关系隐式，给开发者理解和优化带来挑战。尤其对于大型 LLM（大语言模型）推理场景，kernel 的性能直接影响整体系统吞吐量。

### 1.3 项目目标
- **Kernel→Graph**: 将 CUDA kernel 代码转换为结构化的图（Graph）表示
- **Graph→Kernel**: 将图结构反向转换为可编译的 CUDA kernel 代码
- **Graph-UI**: 提供交互式可视化界面，支持人工编辑和探索图结构
- 帮助 LLM 理解 kernel 的计算逻辑和数据依赖关系
- 为 LLM 提供优化建议的结构化输入
- 支持 kernel 性能分析和瓶颈定位

### 1.4 应用场景
| 场景 | 描述 |
|------|------|
| LLM辅助kernel开发 | LLM通过图结构理解kernel逻辑，生成优化建议 |
| kernel代码审查 | 可视化展示计算流程，便于代码审查和知识传递 |
| 性能分析 | 识别计算热点和数据瓶颈 |
| 教学培训 | 帮助初学者理解复杂的并行计算模式 |
| **图驱动开发** | 通过Graph-UI编辑图结构，自动生成kernel代码 |
| **代码重构** | 将现有kernel转换为图，编辑优化后反向生成优化代码 |

## 2. 需求分析

### 2.1 用户角色

| 角色 | 需求描述 |
|------|----------|
| CUDA开发者 | 理解复杂kernel的计算逻辑，获取优化建议 |
| LLM系统集成者 | 将kernel图结构作为prompt输入给LLM |
| 性能工程师 | 分析kernel性能瓶颈 |
| 技术文档编写者 | 生成结构化的kernel分析报告 |
| **可视化用户** | 通过Graph-UI交互式探索和编辑图结构 |
| **代码生成用户** | 通过编辑图结构，自动生成优化的kernel代码 |

### 2.2 功能需求

#### 2.2.1 Kernel解析模块
- 解析CUDA C/C++源码，提取函数定义、变量声明、内存访问模式
- 识别kernel中的计算操作（算术运算、内存操作、同步操作）
- 构建操作之间的数据依赖关系

#### 2.2.2 图结构构建
- 将解析结果转换为有向无环图（DAG）
- 节点表示：操作、变量、内存访问
- 边表示：数据依赖、控制依赖
- 支持多层次图结构（语句级、函数级、模块级）

#### 2.2.3 图可视化
- 生成可阅读的图表示（文本格式、图形格式）
- 支持交互式探索（放大、缩小、高亮路径）
- 生成用于LLM输入的结构化文本描述

#### 2.2.4 分析与优化建议
- 识别计算热点和内存访问模式
- 生成优化建议（如合并内存访问、使用共享内存、循环展开等）
- 支持与LLM的接口，接收优化建议并反馈

#### 2.2.5 Graph→Kernel 代码生成
- 将图结构转换为可编译的CUDA kernel代码
- 支持图节点到代码语句的映射
- 自动生成正确的变量声明和内存访问
- 保持代码可读性和规范性

#### 2.2.6 Graph-UI 交互式界面
- 可视化展示计算图结构
- 支持节点拖拽、添加、删除、编辑
- 实时预览图结构变化
- 支持图结构的导入/导出
- 提供图结构与kernel代码的同步编辑

### 2.3 非功能需求

| 类别 | 需求 |
|------|------|
| 性能 | 支持解析1000+行的kernel文件，响应时间<10秒 |
| 兼容性 | 支持CUDA C/C++ 9.0+语法 |
| 可扩展性 | 支持自定义图节点类型和分析规则 |
| 易用性 | 提供简洁的命令行接口和API |

## 3. 功能规格

### 3.1 核心功能模块

#### 3.1.1 源码解析器
- **输入**: CUDA kernel 源码文件（.cu/.cuh）
- **输出**: 抽象语法树（AST）或中间表示（IR）
- **功能**:
  - 词法分析和语法解析
  - 变量作用域分析
  - 类型推断
  - 内存访问模式识别

#### 3.1.2 图构建引擎
- **输入**: AST/IR
- **输出**: 计算图数据结构
- **功能**:
  - 构建操作节点
  - 建立数据依赖边
  - 识别循环结构和分支
  - 生成多层次图表示

#### 3.1.3 图可视化器
- **输入**: 计算图数据结构
- **输出**: 文本描述 / 图形文件 / JSON结构
- **功能**:
  - 生成用于LLM的结构化prompt
  - 生成Graphviz/DOT格式输出
  - 生成SVG/PNG可视化图片

#### 3.1.4 分析引擎
- **输入**: 计算图
- **输出**: 分析报告 / 优化建议
- **功能**:
  - 计算复杂度分析
  - 内存访问模式分析
  - 并行度评估
  - 生成优化建议列表

#### 3.1.5 代码生成器 (Graph→Kernel)
- **输入**: 计算图数据结构
- **输出**: CUDA kernel 源码
- **功能**:
  - 节点类型到代码语句的映射
  - 变量声明自动生成
  - 内存访问模式转换
  - 代码格式化和美化

#### 3.1.6 Graph-UI 组件
- **输入**: 计算图数据结构
- **输出**: 交互式可视化界面
- **功能**:
  - 图渲染和布局
  - 节点编辑和连接
  - 实时预览
  - 代码同步预览

### 3.2 数据结构设计

#### 3.2.1 图节点类型

| 节点类型 | 描述 | 属性 |
|----------|------|------|
| `OperationNode` | 计算操作 | op_type, operands, result, latency_estimate |
| `MemoryNode` | 内存访问 | access_type(load/store), address_space, size |
| `VariableNode` | 变量/寄存器 | name, type, scope, is_array |
| `ControlNode` | 控制流 | control_type(if/for/while), condition |
| `FunctionNode` | 函数调用 | func_name, arguments, is_kernel |

#### 3.2.2 边类型

| 边类型 | 描述 | 属性 |
|--------|------|------|
| `DataFlow` | 数据依赖 | src_node, dst_node, data_type |
| `ControlFlow` | 控制依赖 | src_node, dst_node, condition |
| `MemoryDependency` | 内存依赖 | src_node, dst_node, memory_order |

#### 3.2.3 图数据结构

```typescript
interface GraphNode {
  id: string;
  type: 'operation' | 'memory' | 'variable' | 'control' | 'function';
  label: string;
  properties: Record<string, unknown>;
}

interface GraphEdge {
  id: string;
  source: string;
  target: string;
  type: 'data' | 'control' | 'memory';
  properties: Record<string, unknown>;
}

interface KernelGraph {
  name: string;
  nodes: GraphNode[];
  edges: GraphEdge[];
  entryPoint: string;
  metadata: {
    totalOperations: number;
    memoryAccessCount: number;
    loopCount: number;
    estimatedFLOPs: number;
  };
}
```

### 4.3 核心模块设计

#### 4.3.1 解析引擎
- **职责**: 将CUDA源码转换为结构化的AST/IR
- **输入**: .cu/.cuh文件路径或源码字符串
- **输出**: 包含语义信息的AST节点树

#### 4.3.2 图构建引擎
- **职责**: 从AST构建计算图
- **核心算法**:
  - 数据流分析：识别变量定义-使用链
  - 控制流分析：识别循环和分支结构
  - 依赖图构建：建立节点间的依赖关系

#### 4.3.3 分析引擎
- **职责**: 对计算图进行性能分析和优化建议生成
- **分析维度**:
  - 计算复杂度（FLOPs估算）
  - 内存访问模式（全局/共享/寄存器）
  - 并行化潜力评估
  - 性能瓶颈识别

## 5. 接口设计

### 5.1 命令行接口 (CLI)

| 命令 | 功能 | 参数 |
|------|------|------|
| `kerg parse` | 解析kernel源码 | `--input`, `--output`, `--format` |
| `kerg visualize` | 生成可视化图 | `--input`, `--output`, `--format` |
| `kerg analyze` | 分析kernel性能 | `--input`, `--output` |
| `kerg prompt` | 生成LLM prompt | `--input`, `--output`, `--style` |
| `kerg generate` | 从图生成kernel代码 | `--input`, `--output` |
| `kerg serve` | 启动API服务 | `--host`, `--port` |

### 5.2 REST API

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/v1/parse` | POST | 解析kernel源码 |
| `/api/v1/graph` | POST | 获取计算图结构 |
| `/api/v1/visualize` | POST | 获取可视化输出 |
| `/api/v1/analyze` | POST | 获取分析报告 |
| `/api/v1/prompt` | POST | 获取LLM prompt |
| `/api/v1/generate` | POST | 从图生成kernel代码 |
| `/api/v1/graph/save` | POST | 保存图结构 |
| `/api/v1/graph/load` | GET | 加载图结构 |

#### 5.2.1 API请求示例

**POST /api/v1/parse**
```json
{
  "source_code": "__global__ void matrixMul(float *A, float *B, float *C, int N) { ... }",
  "options": {
    "include_comments": true,
    "resolve_types": true
  }
}
```

**响应**
```json
{
  "success": true,
  "kernel_name": "matrixMul",
  "parameters": [...],
  "operations": [...],
  "memory_accesses": [...]
}
```

**POST /api/v1/prompt**
```json
{
  "kernel_name": "matrixMul",
  "graph": { /* 计算图结构 */ },
  "style": "detailed"
}
```

**响应**
```json
{
  "success": true,
  "prompt": "分析以下CUDA kernel的计算图...",
  "tokens": 2456
}
```

## 8. 安全性与合规性

### 8.1 安全考虑

| 风险 | 描述 | 缓解措施 |
|------|------|----------|
| 代码注入 | 恶意CUDA代码可能包含危险操作 | 限制解析范围，不执行代码 |
| 资源耗尽 | 超大文件可能导致内存溢出 | 设置文件大小限制，流式处理 |
| 路径遍历 | 恶意路径输入 | 使用安全的路径解析，白名单校验 |
| DoS攻击 | 恶意构造的代码导致解析器崩溃 | 添加超时机制，限制解析时间 |

### 8.2 数据处理
- 源码仅在内存中处理，不持久化存储
- 支持输入验证和清理
- 提供输入大小限制配置

## 9. 附录

### 9.1 输入输出示例

**输入（CUDA kernel）:**
```cuda
__global__ void vectorAdd(float *a, float *b, float *c, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) {
        c[idx] = a[idx] + b[idx];
    }
}
```

**输出（LLM Prompt）:**
```
分析以下CUDA kernel的计算图：

Kernel: vectorAdd
参数: a(global float*), b(global float*), c(global float*), n(int)

计算图结构：
┌─────────────────────────────────────────────────────────────┐
│ 节点列表:                                                   │
│   [V1] idx = blockIdx.x * blockDim.x + threadIdx.x          │
│   [V2] condition: idx < n                                   │
│   [M1] load a[idx] (global memory)                         │
│   [M2] load b[idx] (global memory)                         │
│   [O1] add: a[idx] + b[idx]                                │
│   [M3] store c[idx] (global memory)                        │
│                                                             │
│ 依赖关系:                                                   │
│   V1 → V2 (控制依赖)                                        │
│   V2 → M1, M2 (条件依赖)                                    │
│   M1 → O1 (数据依赖)                                        │
│   M2 → O1 (数据依赖)                                        │
│   O1 → M3 (数据依赖)                                        │
└─────────────────────────────────────────────────────────────┘

请分析此kernel的性能特点并提供优化建议。
```
